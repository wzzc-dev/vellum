## 设计原则

1. **组件化**: UI 是组件树
2. **声明式**: 类似 Flutter 声明式 UI
3. **单向数据流**: 状态变化触发重绘
4. **沙箱安全**: WASM 提供隔离
5. **高性能**: 批量更新

## 核心概念

### Widget (组件)

```moonbit
trait Widget {
  fn build(&self, ctx: &BuildContext) -> Widget;
}
```

### State (状态)

```moonbit
class State<T> {
  value: T;
  fn set(&mut self, value: T);
}
```

### BuildContext (构建上下文)

构建组件时的上下文

### Element (元素树)

内部使用，负责挂载和更新组件树
